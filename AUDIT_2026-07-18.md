# Game Audit — 2026-07-18

Four parallel audits (noise, gameplay/progression, onboarding/clarity, architecture) run against
`main` @ 9d3bc4b. Condensed findings below; fixes shipped on this branch are marked ✅.

## 1. Noise audit — why the game feels noisy

The de-noising scaffolding already in place is good (Decision Inbox triage, toast cap of 3, Calm
Mode, GainFX throttling). The remaining noise is concentrated in a few offenders:

| # | Offender | Status |
|---|---|---|
| 1 | Decision-free earnings "beat" seizes the full screen every quarter (EarningsCall) | ✅ demoted to a toast; miss path (real buyback decision) stays full-screen |
| 2 | Aggregate interrupt ceiling ~17–26 cards/yr — 10 streams share one narrow gap (`minGapWeeks 3/2`) | ✅ widened to 4/3 (~25–33% fewer cards) |
| 3 | BuzzTicker rotates every 4.6s forever + `aria-live` spams screen readers | ✅ 9s rotation, paused when hidden, aria-live off, reframed as editorial wire |
| 4 | ~50 confirmation toasts, no dedup, most echoing already-visible state changes | ✅ dedup added; pure-echo confirmations pruned (Market price/campaign/restock/shares) |
| 5 | Launch payoff stacks ~7 feedback channels; verdict toast duplicates LaunchReveal | ✅ verdict toast removed; confetti/sound/haptics/reveal kept |
| 6 | `hq` nav dot overloaded with ~8 unrelated meanings | Open — split decision-required vs claimable-reward dot styles |
| 7 | GainFX tokens double the HUD number tweens during fast-forward | Open — suppress passive tokens while fast-forwarding |

Determinism note: the pinned 160-week test runs a solo run that raises no opportunistic interrupts,
and the budget tests read `BALANCE.interrupts.*` dynamically, so the gap retune is pin-safe.

## 2. Gameplay & progression audit — what players grind toward

The code is ahead of the roadmap docs: ascension/Heat, Frontier Tech lanes, board mandates +
confidence tiers, Era 5 (Autonomy), Goals Ledger are all shipped. Real gaps:

1. **No long-horizon preview.** Unlocks appear unannounced; a new player cannot see that a 10th
   category, Era 5, IPO, or Frontier Tech exists. Objectives show exactly one next move.
   ✅ addressed by the new Company Roadmap codex + "Coming up" objectives preview.
2. **The ~16-week post-launch sell window is dead time** — little meaningful input between launches.
3. **Maxing is rarely wrong** — no per-project BOM/complexity budget, so the design loop goes
   auto-pilot after the first hit; franchise equity actively rewards re-shipping the same spec.
4. **Eras 1→2 are mechanically identical** (`eraModifiers[0] === [1]`); the first "graduation" reads
   as "same game, new categories."
5. **Endgame grind is numeric, not new verbs** — Legacy Tree / Frontier / perks all funnel through
   the same 4-field `PerkBonus`.

Gaps 2–5 map to the ranked feature list in `FEATURES_RANKED_2026-07-18.md`.

## 3. Onboarding & clarity audit — why the game feels confusing

The first product is beautifully guided (pre-filled starter phone, live-state Coach). Confusion
comes after:

1. **No progressive disclosure** — all 5 tabs and ~40 subsystems reachable from week 1; only 8–10
   are load-bearing in the first hour.
2. **The post-first-launch cliff** — the Coach retires exactly when the game triples in surface
   area; the one-line objective ladder is the only remaining guide.
3. **Unexplained load-bearing vocabulary** — tuning, subsystems, novelty/fatigue, climate,
   aesthetics, buzz, brand awareness, valuation momentum, board confidence, capacity strategy,
   dual-source, verdict tiers have no glossary entries.
4. Three different "fit" numbers collide (headline Fit vs segment fit vs design-brief fit).
5. HQ is an unprioritized scroll of ~14 cards; Factory world toggle sits next to the company name
   from week 1; the glossary is a two-tap detour discovered late.

Recommended sequence: glossary fill → progressive tab reveal (Office+Design first; Research/Market
after first launch; Company after first hire/era 2) → gate heavy Company/Market sections → extend
the first-ship UnlockCard into a map of the tabs → defer the Factory toggle.

## 4. Architecture audit — how to build safely here

- `npm test`: 127 files / 1215 tests green (~41s). Beware: `npm test` exits 0 even if vitest is
  missing — verify "Test Files N passed" in output.
- Salt registry in CLAUDE.md matches code 1:1. Undocumented sub-salts found: **2391/2392/2393**
  (rival-clash, `competitors.ts`) — add to CLAUDE.md. Next fresh salts: **277, 281, 283, 293…**
- The determinism pin covers a do-nothing solo run only; systems gated on player action must ship
  their own replay-twice integration test.
- Cheapest safe extension points: pure engine catalogs (`objectives`, `collections`, `legacyTree`,
  `frontier`, `eventChains`, challenge `MUTATORS`) and cross-run profile stores (`museum`,
  `founderLegend`) — they sidestep the interrupt budget and the sim entirely.
- Biggest tax: `gameState.ts` (6,009 lines) — every new system edits it; consider extracting
  resolvers to `src/state/resolvers/*`. Screens 1.5–2.3k lines each; prefer new sheets over growing
  them. New interrupts compete for an already-contended budget — prefer persistent cards.
